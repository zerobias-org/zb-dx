# PKV (user preferences) API returns 500 on UAT

## What I was trying to do

Persist user filter preferences and UI settings across sessions using the ZeroBias Principal Key-Value (PKV) store, so preferences sync cross-device.

## What happened

PKV API endpoint returns HTTP 500 on UAT. Every save/load call fails silently. Had to implement a dual-backend preference service that always writes to localStorage first, then attempts PKV as a secondary async write that silently fails.

All pre-prod environments (`environment.ts`, `environment.uat.ts`, `environment.vercel.ts`) are hardcoded to `prefsBackend: 'localStorage'` with inline comments explaining the workaround. Only production uses `'pkv'`.

## What I expected

PKV to work consistently across all environments, or at minimum return a useful error message instead of a generic 500.

## Workaround (if any)

Two-tier preference service:
1. Always save to localStorage immediately (synchronous, reliable)
2. Attempt PKV async save only if `prefsBackend === 'pkv'`
3. On PKV failure, silently fall back — localStorage has authoritative copy

Users lose cross-device sync on all non-prod environments.
