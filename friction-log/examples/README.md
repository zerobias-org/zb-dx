# Friction Log Examples

These files are **fictional examples** showing how a friction-log entry evolves through each lifecycle stage. They're here as a tutorial for new contributors who want to see what "done right" looks like.

| File | Stage | What it shows |
|------|-------|---------------|
| `2026-03-15-example-auth-token-refresh.md` | **draft** | A fresh pain point, just logged by one dev |
| `2026-03-20-example-entity-pagination.md` | **confirmed** | A second dev hit the same issue and added their experience |
| `2026-03-25-example-sdk-init-errors.md` | **task-created** | Escalated to a ZB platform task |
| `2026-03-30-example-connection-timeout.md` | **resolved** | The fix shipped |
| `2026-03-10-example-angular-ssr.md` | **promoted** | Graduated into a customer-facing guide |

> These are **NOT** real friction reports. They use fake ZB task IDs, fake SDK behavior, and fake names. They exist only so you can read them and understand the flow.

The `examples/` subfolder is ignored by `/friction list` and `/friction check` — real entries live directly in `friction-log/`.
